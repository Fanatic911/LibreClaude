# Security Policy

## Supported Versions

**We strongly recommend using only version 1.1 of this project.** It is the most stable and secure version available. Any older versions are not supported and their use is highly discouraged.

| Version | Supported          |
| ------- | ------------------ |
| 1.1     | :white_check_mark: |
| < 1.1   | :x:                |

## Reporting a Vulnerability

Security is a top priority for this project. If you discover a vulnerability, please report it privately to ensure it can be addressed before being made public.

### How to report
Please send your vulnerability report directly to: **[your-email@example.com]** (or via a private GitHub Security Advisory).

### What to include
To help us investigate, please provide:
* A clear description of the vulnerability.
* Steps to reproduce the issue.
* Impact of the vulnerability.

### What to expect
* **Acknowledgment:** You will receive an initial response within 48 hours.
* **Updates:** We will provide status updates throughout the investigation process.
* **Resolution:** If the vulnerability is accepted, we will work on a patch and notify you before disclosing it publicly. If declined, we will explain the reasoning.

---
> [!IMPORTANT]
> Do not open a public Issue on GitHub for security vulnerabilities. Public disclosure may put users at risk.
